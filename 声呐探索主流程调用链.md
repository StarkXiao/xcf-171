## 三、目标发现：声呐探测流程

### 3.1 调用链图

```
游戏主循环 (PIXI.Ticker)
    │
    ▼
GameController.update(delta)
    │
    ├─► updatePlayerPosition(delta)       // 更新玩家位置
    │
    ├─► oceanEventSystem.update(delta)    // 更新海洋事件
    │
    └─► sonar.update(delta, targets)      // ⭐ 声呐探测核心
           │
           ├─ 遍历所有声波(waves)
           │    │
           │    ├─  wave.radius += speed * delta  // 声波扩散
           │    ├─  wave.alpha 衰减
           │    └─  声波超出maxRadius → 标记inactive
           │
           ├─ 对每个声波，遍历所有目标(targets)
           │    │
           │    └─ 检测声波与目标的碰撞
           │         │  (dist >= prevRadius - target.radius && dist <= wave.radius + target.radius)
           │         │
           │         ├─【普通目标】creature / wreck
           │         │    │
           │         │    ├─  target.discovered = true
           │         │    ├─  discoveredIds.push(target.id)
           │         │    ├─  discoveredPositions.push(position)
           │         │    │
           │         │    ├─  根据稀有度生成回波点(echos)
           │         │    │   ├─ common: 1-3个
           │         │    │   ├─ rare: 更多数量 + 1.2倍生命
           │         │    │   └─ legendary: 特殊回波 + 回调
           │         │    │
           │         │    └─  传说目标 → onLegendaryEchoGenerated()
           │         │         └─ legendaryChainSystem.onTargetDiscovered()
           │         │
           │         └─【危险目标】danger
           │              │
           │              ├─ 第一阶段：undetected → suspected
           │              │    └─ 生成可疑回波(模糊、半透明)
           │              │
           │              └─ 第二阶段：suspected → confirmed
           │                   └─ 生成确认回波(清晰、完整)
           │
           ├─ 回波点生命周期管理
           │    ├─ echo.life -= delta
           │    └─ echo.alpha = life / maxLife
           │
           ├─ 清理失效的声波和回波
           │
           └─ 返回结果对象 { echos, discoveredTargetIds, discoveredPositions, ... }
    │
    ▼
GameController 处理发现结果
    │
    ├─► scoreSystem.discoverTarget(position)  // 发现目标计分
    │    ├─ state.discoveredTargets++
    │    └─ increaseSonarCombo()              // 连探+1
    │
    ├─► missionSystem.onDiscoverTarget()      // 任务进度
    │
    └─► tutorialSystem.recordAction()         // 教程记录
```

### 3.4 危险目标的两阶段探测

危险目标（danger）采用特殊的两阶段探测机制：

| 阶段 | 触发条件 | 视觉表现 | 可收集性 |
|------|---------|---------|---------|
| **undetected** | 初始状态 | 完全不可见 | - |
| **suspected** | 第一次被声呐扫到 | 模糊问号、虚线圈、黄褐色调 | 不可收集 |
| **confirmed** | 第二次被声呐扫到 | 完整危险标记 | 可收集（但会扣血） |

## 四、积分结算：目标收集流程

### 4.1 调用链图

```
用户点击目标
    │
    ▼
App.vue: processInteraction
    │
    ▼
GameController.handleTap(worldPos)
    │
    ├─ 先尝试收集宝藏事件
    │    └─ oceanEventSystem.tryCollectTreasure()
    │
    └─ 遍历所有目标，检测点击命中
         │  (dist <= target.radius + 15 && target.discovered)
         │
         ├─ 可疑危险目标 → 跳过（不可点击）
         │
         └─ 命中目标
              │
              ├─ target.collected = true
              ├─ collectedCount++
              │
              ├─ 计算分数修正
              │    ├─ getScoreModifier(position)  // 宝藏事件×1.2
              │    ├─ 深度奖励 (沉船额外加成)
              │    └─ 深度区域分数倍率
              │
              ├─► scoreSystem.collectTarget(target)  // ⭐ 核心积分
              │    │
              │    ├─【危险目标】
              │    │    ├─ state.lives -= lifePenalty
              │    │    ├─ state.score += scorePenalty (负值)
              │    │    ├─ breakCombo() / breakSonarCombo()  // 断连
              │    │    └─ onScoreEvent({ type: 'damage' })
              │    │
              │    └─【普通目标】creature / wreck
              │         ├─ increaseCombo()               // 连击+1
              │         │    ├─ state.combo++
              │         │    ├─ state.comboMultiplier 更新
              │         │    ├─ 检查连击里程碑奖励
              │         │    └─ onComboEvent()
              │         │
              │         ├─ 计算最终分数
              │         │    points = basePoints × comboMultiplier × typeMultiplier
              │         │    (typeMultiplier: creature=1.0, wreck=1.5)
              │         │
              │         ├─ state.score += adjustedPoints
              │         │
              │         └─ onScoreEvent({ type: 'collect' })
              │
              ├─► missionSystem.onCollectTarget()     // 任务进度
              │
              ├─► legendaryChainSystem.onTargetCollected()  // 传说链路
              │
              ├─► collectionSystem.recordTarget()      // 图鉴记录
              │    └─ 首次发现 → onUnlock() 事件
              │
              ├─ 检查升级 checkLevelUp()
              │    │
              │    ├─ 收集数达标 → level++
              │    ├─ state.sonarCharges += 2
              │    ├─ state.score += levelBonus
              │    ├─ targetGenerator.generateTargets()  // 生成新目标
              │    └─ onLevelUp()
              │
              ├─ 检查死亡
              │    └─ lives <= 0 → 游戏结束
              │         ├─ scoreSystem.endGame()
              │         └─ onGameOver(finalScore)
              │
              └─ 返回 { hit: true, target }
```

### 4.2 关键代码位置

- **目标点击处理**: `src/game/GameController.ts:618-716`
- **积分结算核心**: `src/game/ScoreSystem.ts:453-514`
- **连击系统**: `src/game/ScoreSystem.ts:400-421`
- **升级检查**: `src/game/ScoreSystem.ts:593-613`

### 4.3 积分计算公式

```
最终得分 = 基础分数 × 连击倍率 × 类型倍率 × 深度倍率 × 事件倍率

其中：
- 基础分数：由目标类型和稀有度决定
- 连击倍率：comboMultiplier（随连击数递增）
- 类型倍率：creature=1.0, wreck=1.5
- 深度倍率：越深越高，每深一区+0.5倍
- 事件倍率：宝藏区域×1.2
```

### 4.4 状态变化

| 状态字段 | 变化 | 所属系统 |
|---------|------|---------|
| `target.collected` | false → true | 目标对象 |
| `score` | +调整后的分数 | ScoreSystem |
| `lives` | 危险目标时减少 | ScoreSystem |
| `combo` | 普通目标+1，危险目标清零 | ScoreSystem |
| `comboMultiplier` | 根据连击数更新 | ScoreSystem |
| `level` | 达标时+1 | ScoreSystem |
| `discoveredTargets` | - | ScoreSystem |

## 七、总结

### 7.1 核心设计模式

1. **MVC 架构**：GameController 作为控制器，协调 Model（ScoreSystem、SonarSystem 等）与 View（MapRenderer、Vue组件）
2. **观察者模式**：通过回调函数（onStateChange、onScoreEvent 等）实现状态变更通知
3. **组件化设计**：各系统（SonarSystem、ScoreSystem、TargetGenerator）职责单一，相互独立

### 7.2 数据流方向

```
用户输入 → Controller → 各业务系统 → 状态更新 → 回调通知 → UI渲染
                        ▲
                        │
                   游戏循环驱动
                   （每帧更新）
```

### 7.3 关键状态清单

| 状态类别 | 存储位置 | 渲染方式 |
|---------|---------|---------|
| 游戏数值（分数、生命、等级等） | ScoreSystem.state | Vue响应式 + GameHUD |
| 声呐波 | SonarSystem.waves | PIXI每帧重绘 |
| 回波点 | SonarSystem.echoPoints | PIXI每帧重绘 |
